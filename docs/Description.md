Cosmic Bliss is an adult game in Godot 4.6 built around high-fidelity, physics-driven tentacle interactions with a single hero character. The design bet is that interaction feel — not content volume — carries the game: every encounter produces unique emergent moments rather than replaying scripted scenes. The tone reaches from broken hopelessness to transcendent bliss, with a surreal/psychedelic/cosmic aesthetic rather than grimdark or horror.
The experience rests on a few pillars. Tentacles are always alive: they breathe, drift, and writhe even without player input, and never freeze. Interactions are simulation outcomes, not animations — identical-looking setups never play out the same way because shape, friction, wetness, angle, and timing all mix. The hero is not a reactive mannequin; she carries short-term state (aroused, in pain, ecstatic, afraid) on top of a slower-moving mindset (broken↔blissful, resistant↔yielding, alert↔dulled, tender↔feral), and that state is coupled bidirectionally to physics: yielding makes grip easier, tension makes it harder, arousal drives wetness, all in one loop. Player control feels like nudging a living thing — specific tentacles can be taken over while the rest of the system continues underneath, and releasing control fades seamlessly back into autonomy.
Four custom Godot extensions do the work. They are being built before any gameplay is designed — the wager is that if the interaction itself is compelling enough, loops will follow.
TentacleTech is the tentacle physics and orifice system. A single PBD (Position-Based Dynamics) solver handles length, bending, collision, friction, and attachment — no mixing of Verlet, IK, springs, or a traditional rig. Tentacle shape is a solver output, not something an animation system fights against. Each tentacle is a chain of 16–48 particles on CPU; thousands of mesh vertices are skinned to that spline in a vertex shader. Orifices are driven ring-bone clusters (8 radial bones per opening) plus EntryInteraction state tracking persistent tentacle/orifice contacts — multi-tentacle, bilateral-compliance aware. Volume preservation and directional asymmetry (from ring pressure) propagate along the spline post-solve and compose multiplicatively in the shader. Mechanical sound — squelch, friction slide, ring creak, stick-slip chirp, fluid strand separation — is driven from physics events directly, so audio never lags the simulation. The extension also owns the stimulus bus: a single autoload publishing events, continuous channels, and a modulation channel that other systems (primarily Reverie) write back into.
Tenticles is a fully custom GPU particle system, modeled on Housemarque's NGP architecture from the Returnal GDC 2022 talk. It bypasses Godot's GPUParticles3D entirely — indirect draw into a compute-written vertex buffer via a CompositorEffect, GPU-side spatial hashing via counting-sort, a resource registry so particles can read external data textures by name. Tentacle rendering is the first vertical slice, and the same system will later drive slime, fluid sim, and destructible debris. The compute-shader-is-the-effect model is explicit: a .udon.glsl file with annotation-driven parameters is the authored effect; inspector UI is a reflection of the shader, not a source of truth.
Marionette is the active ragdoll — the hero's body itself. It ships pure GDScript (SPD muscle controller via _integrate_forces, single skeleton, PhysicalBoneSimulator3D.influence = 1.0) with a C++ port held in reserve only if profiling demands it. Two vocabularies coexist: anatomical (flexion, abduction, medial rotation) in the public API and authored resources, joint-axis internally. The anatomical frame is baked into physics at ragdoll creation, so runtime has zero geometric work. An 84-bone custom MarionetteHumanoidProfile includes toe bones. Poses, cyclic animations (breathing, shiver, tremor), emotional body overlays, and motion macros compose additively on top of SPD-driven bones. A per-bone strength parameter takes the character continuously from limp ragdoll to actively held pose, and contact-triggered alpha reduction kills wall vibration.
Reverie is the reaction/emotion layer and the slowest-moving of the four. It reads the stimulus bus at ~20 Hz, holds a state distribution (Aroused, Ecstatic, In-pain, Curious, Hopeless, Neutral, etc.) plus a 6-axis continuous mindset vector that updates over minutes-to-hours. Reaction profiles translate (event type + state + mindset) into facial blendshapes, skin shader parameters (flushing, sweat, tear tracks), vocalization queues, Marionette pose targets, and — critically — modulation writes back into TentacleTech: grip strength multipliers, stretch stiffness, orifice seek intensity, voluntary body motion. That write closes the physics-to-emotion loop: the same stimulus on a broken-resistant character vs. a blissful-yielding one produces measurably different physics on the next tick. Reverie is spec'd in Reverie_Planning.md but deferred until TentacleTech's bus is stable.
The composition is the point. Physics runs the 60 Hz tick, emits events and continuous state onto the bus, and reads modulation. Reverie ticks slower, reads events + state, writes modulation and drives face/voice/pose intent. Marionette holds the body up (or lets it fall) with SPD torques, takes pose targets from Reverie, and provides the skeleton that tentacles collide against. Tenticles handles high-count GPU visuals and future fluid sim. Each system has a clean contract with the others; none of them reaches into another's internals. The hero being "alive" falls out of three things layered: physics-native noise on continuous parameters, Reverie state blending, and Marionette cyclic overlays — never from a behavior tree deciding to seem alive.
